Критерии оценки выполнения задания
Оценка работы производится по совокупности технических и архитектурных критериев. Максимальное количество баллов за задание — 50. Итоговая оценка складывается из суммы баллов по каждому разделу. Работа, не соответствующая базовым архитектурным требованиям, не может быть оценена выше 25 баллов независимо от корректности отдельных частей.
1. Структура проекта и архитектура приложения — до 14 баллов
Оценивается корректность организации проекта и соблюдение принципов разделения ответственности между слоями приложения.
Корректно создана структура директорий app/, core/, db/, schemas/, repositories/, services/, usecases/, api/.
— 4 балла
Слои приложения логически разделены, каждый файл находится в своей зоне ответственности и не дублирует функциональность других слоёв.
— 4 балла
В эндпоинтах отсутствует бизнес-логика и прямое взаимодействие с базой данных, вся логика вынесена в usecase и repository слои.
— 4 балла
Использование dependency injection через Depends для передачи usecase и репозиториев реализовано корректно.
— 2 балла
2. Работа с uv и окружением проекта — до 6 баллов
Оценивается корректность настройки проекта и управление зависимостями.
Проект инициализирован с использованием uv, виртуальное окружение создаётся командой uv venv.
— 2 балла
Зависимости корректно описаны в pyproject.toml, установка выполняется через uv pip compile и uv pip install.
— 2 балла
Проект успешно запускается в чистом окружении после выполнения инструкций из README без ручной установки библиотек.
— 2 балла
3. Реализация аутентификации и авторизации (JWT) — до 10 баллов
Оценивается корректность реализации безопасности и работы с токенами.
Реализован эндпоинт регистрации пользователя с сохранением email и хешированного пароля в базе данных.
— 4 балла
Реализован эндпоинт логина пользователя с выдачей JWT access token.
— 3 балла
Защищённые эндпоинты недоступны без токена, корректно используется авторизация через кнопку Authorize в Swagger.
— 3 балла
4. Интеграция с LLM через OpenRouter — до 10 баллов
Оценивается корректность взаимодействия с внешним API и обработка ответа модели.
Реализован отдельный сервис для взаимодействия с OpenRouter, не связанный напрямую с API-слоем и базой данных.
— 4 балла
Корректно формируется запрос к LLM с использованием модели stepfun/step-3.5-flash:free.
— 3 балла
Ответ от LLM корректно возвращается пользователю через эндпоинт POST /chat.
— 2 балла
Ошибки внешнего сервиса корректно обрабатываются и не приводят к падению приложения.
— 1 балл
5. Работа с базой данных и историей диалога — до 10 баллов
Оценивается корректность хранения и извлечения данных.
Реализовано сохранение запросов пользователя и ответов модели в базе данных SQLite.
— 4 балла
Эндпоинт GET /chat/history возвращает историю диалога текущего пользователя.
— 4 балла
История корректно привязана к пользователю, данные разных пользователей не смешиваются.
— 2 балла
6. Документация и демонстрация работы
Оценивается качество документации и подтверждение работоспособности проекта.
Файл README.md содержит полное описание установки, настройки и запуска проекта через uv.  
В README приложены скриншоты работы каждого эндпоинта, включая:
регистрацию пользователя,
логин и получение JWT,
авторизацию через Swagger,
вызов POST /chat,
получение истории через GET /chat/history,
удаление истории через DELETE /chat/history,
скриншоты демонстрируют использование email формата student_surname@email.com при регистрации.
В случае отсутствия скриншотов работы эндпоинтов в README работа не принимается.
При регистрации должен использоваться email формата student_surname@email.com. Данный email должен быть явно указан на скриншотах в README. При невыполнении этого работа так же не принимается.
Итоговая шкала оценки
50–45 баллов — задание выполнено полностью и корректно, архитектура соответствует требованиям, проект готов к расширению.
44–35 баллов — задание выполнено в целом корректно, имеются незначительные архитектурные или оформительские недочёты.
34–30 баллов — задание выполнено частично, имеются нарушения архитектуры или отсутствуют ключевые элементы.
Менее 30 баллов — задание выполнено некорректно, основные требования не соблюдены.
