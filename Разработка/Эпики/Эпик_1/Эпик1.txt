## Эпик 1: Инфраструктура и конфигурация

### Цель

Создать базовую структуру проекта с управлением зависимостями через `uv` и конфигурацией через Pydantic Settings.

### Технический дизайн

1. **Инструменты:** `uv` для управления зависимостями, `ruff` для линтинга.
2. **Структура:** Модульная структура `app/` с подпапками по слоям.
3. **Конфигурация:** Чтение `.env` через `pydantic-settings`, единый объект `settings`.

### Задача 1.1: Инициализация проекта и зависимости

**Acceptance Criteria:**
- [ ] `uv init` выполнен, создан `pyproject.toml`
- [ ] Зависимости из ТЗ добавлены в `pyproject.toml`
- [ ] `uv venv` создаёт виртуальное окружение
- [ ] `uv pip compile` и `uv pip install` работают без ошибок
- [ ] `ruff check` проходит на пустом проекте

**Файлы:**
- `pyproject.toml`
- `README.md`
- `.gitignore`

### Задача 1.2: Создание файловой структуры

**Acceptance Criteria:**
- [ ] Созданы директории: `app/`, `app/core/`, `app/db/`, `app/schemas/`, `app/repositories/`, `app/services/`, `app/usecases/`, `app/api/`
- [ ] В каждой директории создан `__init__.py`
- [ ] Создан `app/main.py` (точка входа)

**Файлы:**
- Все `__init__.py` в указанных директориях

### Задача 1.3: Конфигурация приложения

**Acceptance Criteria:**
- [ ] Класс `Settings` в `app/core/config.py` читает переменные из `.env`
- [ ] Поля конфигурации: `APP_NAME`, `ENV`, `JWT_SECRET`, `JWT_ALG`, `ACCESS_TOKEN_EXPIRE_MINUTES`, `SQLITE_PATH`, `OPENROUTER_API_KEY`, `OPENROUTER_BASE_URL`, `OPENROUTER_MODEL`, `OPENROUTER_SITE_URL`, `OPENROUTER_APP_NAME`
- [ ] Создан `.env.example` с примерами переменных
- [ ] `settings` — единственный экземпляр, импортируемый в других модулях

**Файлы:**
- `app/core/config.py`
- `.env.example`
