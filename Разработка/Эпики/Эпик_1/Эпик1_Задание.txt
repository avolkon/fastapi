# 🎯 Задание на разработку: EPIC1-TASK1.1 Инициализация проекта и зависимости (uv)

## Контекст
Первый шаг эпика 1: зафиксировать зависимости по ТЗ и воспроизводимость через **uv**.

## Текущее состояние
**Файл:** `pyproject.toml`, корень репозитория  
**Функция/класс:** метаданные проекта и dependencies  
**Текущий вывод/поведение:** пакеты не совпадают с перечнем ТЗ или установка хрупкая  
**Необходимый вывод/поведение:** `uv pip compile`/`install` выполняются без ошибок; `ruff check` есть в зависимостях и проходит

## Требования ТЗ
Из ТЗ_0: «Перед тем, как выполнить установку необходимо скопировать содержимое в ваш pyproject.toml» и использование uv для окружения.

## Задача
Настроить `pyproject.toml` и установку зависимостей через **uv** согласно ТЗ.

## Технические требования

### 1. Изменения в pyproject и окружении
**Логика работы:**
1. Заполнить `[project]` и `dependencies` как в блоке из ТЗ_0.
2. Обеспечить сценарий: `uv venv` → установка через `uv pip compile`/`uv pip install` (как в ТЗ).
3. Добавить `ruff`; прогон `uv run ruff check` на доступных файлах без ошибок.

### 2. Формат вывода
```
Команды завершения без traceback:
uv / ruff ок
```

**Где:**
- `uv`/`ruff` — успешное выполнение и отсутствие ошибок линтера на текущем дереве файлов.

### 3. Обработка ошибок
```python

Добавить минимальный набор модулей на следующих задачах или ограничить scope `check`.
```

# 🎯 Задание на разработку: EPIC1-TASK1.3 Конфигурация приложения (`Settings`)

## Контекст
Центральный объект настроек из `.env` нужен всем следующим слоям (JWT, SQLite, OpenRouter).

## Текущее состояние
**Файл:** `project/app/core/config.py`, `.env.example`  
**Функция/класс:** `Settings`, `settings` singleton  
**Текущий вывод/поведение:** переменные разбросаны или не читаются из `.env`  
**Необходимый вывод/поведение:** единый `Settings(pydantic-settings)` с полями из ТЗ; `settings` импортируется всем приложением

## Требования ТЗ
ТЗ_0 для `core/config.py`: класс Settings, чтение `.env`, перечень полей (APP_NAME, ENV, JWT, SQLITE_PATH, OpenRouter и т.д.).

## Задача
Реализовать конфигурацию через pydantic-settings и образец `.env.example`.

## Технические требования

### 1. Настройки окружения
**Логика работы:**
1. Описать поля: `APP_NAME`, `ENV`, `JWT_*`, `ACCESS_TOKEN_EXPIRE_MINUTES`, `SQLITE_PATH`, `OPENROUTER_*`.
2. Создать единственный экземпляр `settings` для импорта.
3. Добавить `.env.example` без секретов (плейсхолдеры).

### 2. Формат вывода
```
Пример переменной:
APP_NAME=llm-p
SQLITE_PATH=./app.db
OPENROUTER_MODEL=stepfun/step-3.5-flash:free
```

**Где:**
- значения по умолчанию и описание — только в `.env.example` и документации, секреты не в git.

### 3. Обработка ошибок
```python
# Отсутствует обязательное поле
Pydantic validation error при старте — ожидаемо до заполнения .env локально.
```

## Файлы для изменения
- `project/app/core/config.py`
- .env.example
- **Не изменять:** файлы с реальными секретами `.env`
