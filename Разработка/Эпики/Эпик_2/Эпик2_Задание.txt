# 🎯 Задание на разработку: EPIC2-TASK2.1 Async engine и сессии SQLAlchemy

# 🎯 Задание на разработку: EPIC2-TASK2.2 ORM-модели User и ChatMessage

## Контекст
Модели — контракт хранения пользователей и истории диалогов.

## Текущее состояние
**Файл:** `app/db/models.py`  
**Функция/класс:** `User`, `ChatMessage`, relationships, FK  
**Текущий вывод/поведение:** нет таблиц/полей под ТЗ или неверная связь  
**Необходимый вывод/поведение:** поля как в ТЗ; FK `chat_messages.user_id` с `ondelete=CASCADE`; `created_at` через UTC-safe default

## Требования ТЗ
ТЗ_0 `models.py`: User (email unique indexed), ChatMessage с FK на users; relationship.

## Задача
Реализовать ORM-модели и связь один-ко-многим.

## Технические требования

### 1. Модели данных
**Логика работы:**
1. `User`: id, email (unique/index), password_hash, role, created_at.
2. `ChatMessage`: id, user_id FK → users.id (CASCADE при удалении пользователя — в ORM), role, content, created_at.
3. `relationship` с `User.messages`.
4. Для `created_at`: `datetime.now(timezone.utc)` или `server_default` по команде — не `datetime.utcnow`.

### 2. Формат вывода
```
SQLAlchemy-сущности согласованы с архитектурной таблицей «Схема данных».
```

**Где:**
- email уникален; сообщения принадлежат ровно одному user_id.

### 3. Обработка ошибок
```python
# Неверный тип данных
Привести модель к типам SQLite/SQLAlchemy 2 совместимо.
```
