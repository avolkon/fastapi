# 🎯 Задание на разработку: EPIC3-TASK3.1 Хеширование паролей (passlib/bcrypt)

## Контекст
Security-утилиты для регистрации и логина.

**Файл:** `app/core/security.py` (функции hash/verify часть файла)  
**Функция/класс:** `hash_password`, `verify_password`  
**Текущий вывод/поведение:** функций нет или нестабильные параметры bcrypt  
**Необходимый вывод/поведение:** стабильные функции через passlib; версия bcrypt согласована с ТЗ

## Требования ТЗ
ТЗ_0 для `security.py`: passlib bcrypt; файл не знает роутов и не ходит в БД.

## Задача

## Технические требования

### 1. Password hashing
**Логика работы:**
1. `hash_password(plain: str) -> str`.
2. `verify_password(plain, hashed) -> bool`.
3. Использовать настройки/алгоритм из методички (bcrypt версии из зависимостей).

### 2. Формат вывода
```
verify_password(...) -> True/False без исключений при типовом «неверный пароль».
```

**Где:**
- хеш только для сохранения в `password_hash` в use case/register.

# 🎯 Задание на разработку: EPIC3-TASK3.2 JWT — создание и декодирование

## Контекст
Токен с полями `sub`, `role`, `iat`, `exp` — требование ТЗ для `/auth/me` и защищённых маршрутов.

## Текущее состояние
**Файл:** `app/core/security.py`  
**Функция/класс:** `create_access_token`, `decode_token`  
**Текущий вывод/поведение:** нет поддержки полного payload или некорректная валидация  
**Необходимый вывод/поведение:** JWT выпускается и декодируется с проверкой подписи и срока; используются `JWT_SECRET`, `JWT_ALG`

## Требования ТЗ
ТЗ: payload содержит sub (user id), role, exp, iat.

## Задача
Реализовать выпуск и разбор JWT в `security.py`.

## Технические требования

### 1. JWT утилиты
**Логика работы:**
1. Формировать payload: `sub` как str(user_id), `role`, `iat`, `exp` (exp из TTL настроек).
2. `decode_token` валидирует подпись и exp.
3. Не использовать БД внутри этих функций.

### 2. Формат вывода
```
Payload пример:
{"sub":"1","role":"user","iat":...,"exp":...}
```

**Где:**
- TTL из настроек `ACCESS_TOKEN_EXPIRE_MINUTES`.

### 3. Обработка ошибок
