# 🎯 Задание на разработку: EPIC4-TASK4.1 Клиент OpenRouter (`httpx`)

## Контекст
Интеграция LLM изолирована в сервисе; ошибки становятся `ExternalServiceError`.

## Текущее состояние
**Файл:** `app/services/openrouter_client.py`  
**Функция/класс:** `OpenRouterClient.chat_completion`  
**Текущий вывод/поведение:** нет клиента или сырой проброс ошибок без доменной обёртки  
**Необходимый вывод/поведение:** POST chat/completions с заголовками из settings; timeouts; текст ответа; ошибки провайдера → ExternalServiceError

## Требования ТЗ
ТЗ: Authorization, HTTP-Referer, X-Title; модель из настроек; доменная ошибка внешнего сервиса.

## Задача
Реализовать асинхронный клиента OpenRouter по tech design задаче 4.1.

## Технические требования

### 1. OpenRouter HTTP-клиент
**Логика работы:**
1. Собирать URL из `OPENROUTER_BASE_URL` + `/chat/completions`.
2. Заголовки: Bearer key, Referer/App name по настройкам.
3. Тело: model из настроений, список messages, temperature параметр метода.
4. Таймаут ~60 сек (как в дизайне).
5. HTTP ошибки и битые ответы → `ExternalServiceError`.

### 2. Формат вывода
```
Метод возвращает текст ответа ассистента (str).
При ошибке — доменное исключение без падения приложения.
```

**Где:**
- модель по умолчанию критерия: `stepfun/step-3.5-flash:free` через settings.

### 3. Обработка ошибок
```python
# 4xx/5xx или нет choices[0].message.content
ExternalServiceError с безопасным сообщением клиенту (детали в лог опционально).
```
