## Эпик 7: Документация, тестирование, приёмка

### Цель

Подготовить проект к сдаче: документация, скриншоты, проверка качества.

### Задача 7.1: README и скриншоты (критично для сдачи)

**Acceptance Criteria:**
- [ ] README содержит:
  - Описание проекта
  - Требования (Python 3.11+)
  - Установку через **`uv`** (по **`ТЗ_0`**: `uv venv`, цепочка **`uv pip compile`** / **`uv pip install`**; допустим также описанный в README вариант с **`pip install -e .`**, если он воспроизводимо поднимает зависимости)
  - Настройку `.env` (пример)
  - Запуск: `uv run uvicorn app.main:app --reload`
  - Ссылку на Swagger: `http://localhost:8000/docs`
- [ ] Скриншоты (в README):
  - Регистрация пользователя с email **`student_surname@email.com`**
  - Логин и получение JWT токена
  - Авторизация через кнопку Authorize в Swagger
  - `POST /chat` с запросом и ответом LLM
  - `GET /chat/history` с историей диалога
  - `DELETE /chat/history` (успешное удаление)

**Файлы:**
- `README.md`
- Папка `screenshots/` (если скриншоты отдельно)

### Задача 7.2: Проверка качества кода

**Acceptance Criteria:**
- [ ] **`ruff check`** (по областям приложения или репозитория) не выдаёт ошибок — **обязательно для критериев приёмки**
- [ ] **`ruff format`** прогнан по коду (**рекомендуется**, в критериях балльно главное проверить **`check`**)
- [ ] Нет `print()` в production-коде
- [ ] Нет закомментированного кода в финальной версии

**Команда:**
```bash
uv run ruff check app/
uv run ruff format app/
```

### Задача 7.3: Ручное тестирование сценариев

**Acceptance Criteria:**
- [ ] Приложение запускается без ошибок
- [ ] Swagger UI доступен на `/docs`
- [ ] Регистрация создаёт запись в `app.db` (проверить через `sqlite3`)
- [ ] Логин возвращает JWT (проверить через `jwt.decode`)
- [ ] `POST /chat` с валидным токеном возвращает ответ от LLM
- [ ] `GET /chat/history` показывает отправленные сообщения и ответы
- [ ] `DELETE /chat/history` очищает историю
- [ ] Без токена защищённые эндпоинты возвращают 401

**Инструменты:**
- `curl` или Swagger UI
- `sqlite3 app.db` для проверки БД
